# AlphaConsole 核心流程
## 1. 总体原则
当前系统不是“来一条提醒就打印一次”，而是：
1. 先定义固定发行时点
2. 各 App 在各自的准备时间准备内容
3. 到发行时点后，由各 App 发布 Block
4. 系统把有效 Block 组装成一期 Issue
5. 打印这一期

因此主轴是：**刊次驱动，而不是单消息驱动**。

---

## 2. 定时刊次流程
## 2.1 触发条件
当某个 `PublicationSlot` 到达其发行时间时，系统开始组装一次 `Issue`。

## 2.2 流程步骤
1. 创建本期 `Issue`
2. 生成 `IssueHeader`
3. 收集所有归属于该 `PublicationSlot` 的已启用 App
4. 逐个调用 App 的 `publish()`，每个 App 最多返回一个 Block
5. 过滤掉：
   - `None`
   - 已过期 Block
6. 将剩余 Block 按 FIFO 排列
7. 生成最终 `Issue`
8. 进入打印动作（当前阶段只到组刊，不实现真实打印）

## 2.3 结果
本期 `Issue` 结构为：
- `IssueHeader`
- `Block[]`

---

## 3. 内容准备与出刊分离
## 3.1 原则
App 的内容准备时间，不等于它的出刊时间。

## 3.2 例子
例如“科技新闻” App：
- 08:00 开始准备内容
- 归属 12:00 午刊

这意味着：
- 08:00：开始收集或整理
- 12:00：如已准备好，则发布一个 `NewsBlock`
- 12:00：如仍未准备好，则该 Block 过期，不进入午刊

## 3.3 当前实现要求
当前阶段可以只保留“准备时间”这个语义，不强制实现真实 prepare runner。  
但代码结构必须允许未来补上这个阶段。

---

## 4. SceneApp 发布流程
## 4.1 SceneApp 的角色
`SceneApp` 是一个普通 App 的具体实现。  
它在流程上不特殊，只是产出的 Block 结构更丰富。

## 4.2 SceneApp 的输出
在某次出刊时，`SceneApp` 应发布一个 `SceneBlock`。
`SceneBlock` 支持以下三种有效形式：
1. 只有 note
2. 只有 checklist
3. note + checklist

## 4.3 SceneBlock 内容结构
- 标题：场景名
- note：可选
- checklist：可选
- body：plain-text fallback

---

## 5. 即时独立一期流程
## 5.1 原则
外部即时触发的内容默认不参与当前默认刊次合并，而是生成独立一期。

## 5.2 流程步骤
1. 接收即时触发请求
2. 创建独立的 `Issue`
3. 生成 `IssueHeader`
4. 由触发 App 发布一个 Block
5. 将该 Block 放入本期 Issue
6. 不与当前默认刊次合并
7. 进入打印动作（当前阶段不实现真实打印）

## 5.3 队列语义
当前阶段：
- 不考虑中断
- 不考虑抢占
- 不定义复杂优先级
- 只保留“独立一期”的产品语义

---

## 6. 过期流程
## 6.1 规则
过期判断以 **Block** 为单位。

## 6.2 行为
- 某个 Block 过期时，只删除该 Block
- 其他未过期 Block 仍可进入本期
- 不因为一个 Block 过期而取消整个 Issue
- 错过时机的 Block 不补打

## 6.3 例子
- 08:00 开始准备内容
- 09:00 目标出刊
- 到 09:00 内容还未准备完成

则：
- 对应 Block 过期
- 该 Block 不进入 09:00 这一期

---

## 7. FIFO 组装规则
## 7.1 当前阶段规则
当前阶段默认使用 FIFO 排列内容块。

## 7.2 说明
这里的 FIFO 指：
- 哪个 Block 先进入组刊队列，哪个就先排在前面
- 若代码实现需要稳定顺序，可使用：
  - `created_at`
  - 再加稳定插入顺序

## 7.3 当前阶段不实现
- 复杂优先级
- 多维排序
- 按内容类型分区
- 手工拖拽排序

---

## 8. 开环打印原则
打印行为是开环的。
这意味着：
- 不记录“纸上是否完成”
- 不记录“checklist 是否被勾选”
- 不因为用户是否执行而影响后续内容生产
- 打印只是交付，不是状态回流入口

---

## 9. 历史与记录
当前阶段保留三类记录命名：
- `BlockHistory`
- `IssueHistory`
- `PrintRecord`

但当前阶段不实现复杂历史查询。  
只要求代码结构不把三者混成一个对象。

---

## 10. 当前阶段不进入的流程
以下流程当前必须明确跳过：
- 真实调度器调用
- 真实打印机调用
- 失败重试
- 重印
- 历史回放
- 用户反馈闭环
